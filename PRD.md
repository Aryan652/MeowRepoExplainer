# Product Requirements Document (PRD) - RepoMind AI

## 1. Product Overview
**RepoMind AI** is an AI-powered developer tool designed to help developers, engineering managers, and architects quickly understand and analyze any codebase. By ingesting a repository (via GitHub URL or ZIP upload), RepoMind runs multi-agent AI analysis to auto-generate documentation, map architecture, evaluate technical debt, and allow conversational querying of the codebase.

## 2. Target Audience
- **Software Engineers:** Onboarding to new projects or debugging unfamiliar legacy codebases.
- **Engineering Managers / Tech Leads:** Getting high-level overviews of code quality, technical debt, and team progress.
- **System Architects:** Visualizing module dependencies and infrastructure topography.

## 3. Tech Rules (Tech Stack Definition)

Defining the technical rules and the technology stack is a foundational step that dictates the tools, frameworks, languages, and platforms to be used throughout the project. This ensures consistency, compatibility, and scalability.

### 3.1 Frontend & Framework
- **Framework:** TanStack Start (Full-stack React framework).
- **Routing:** TanStack React Router (`@tanstack/react-router`).
- **State Management:** TanStack Query (`@tanstack/react-query`) for remote data synchronization.
- **UI Library:** React 19, customized Radix UI components (`@radix-ui/*`).
- **Styling Rules:** Vanilla CSS + Tailwind CSS v4. Use predefined utility classes, `clsx`, and `tailwind-merge` for conditional styling. Focus on modern, dynamic aesthetics (e.g., glassmorphism, dynamic animations).
- **Icons & Visuals:** Lucide React for consistent iconography; Recharts for data visualization.

### 3.2 Build & Tooling
- **Package Manager:** Bun (`bunfig.toml`, `bun.lock`). It is strictly required to use Bun for dependency resolution to ensure reproducibility and fast builds.
- **Bundler:** Vite (`vite.config.ts`, `@vitejs/plugin-react`).
- **Linting & Formatting:** ESLint v9, Prettier. All new code must adhere to the existing `.prettierrc` and `eslint.config.js` rules.
- **Language:** TypeScript v5. Strict type-checking is enforced across the codebase.

### 3.3 Deployment & Infrastructure
- **Deployment Platform:** Cloudflare Pages/Workers.
- **Configuration:** Maintain Cloudflare-specific settings in `wrangler.jsonc`.
- **Plugin Requirement:** Use `@cloudflare/vite-plugin` for Vite builds targeting Cloudflare environments.

## 4. Core Features & Capabilities

### 4.1 Marketing & Landing
- **Hero & Value Proposition:** Immediate explanation of the value ("Understand any codebase in minutes").
- **Feature Showcase:** Showcases of the chat interface, architecture maps, and automated documentation.

### 4.2 Dashboard & Workspace
- **Repository List:** Displays all analyzed repositories belonging to the user/workspace.
- **High-level Metrics:** Shows the repository status (Ready, Analyzing, Error), health score, file count, and last updated time.
- **Search:** Search across all analyzed repositories.

### 4.3 Analysis Engine (Flow)
- **Inputs:** Supports public GitHub URLs and `.zip` file uploads (up to 500 MB).
- **Processing Steps:**
  1. Cloning repository
  2. Detecting languages & frameworks
  3. Parsing AST (Abstract Syntax Tree) & extracting symbols
  4. Computing embeddings for semantic search
  5. Running multi-agent analysis (Documentation Agent, Security Agent, Architecture Agent, Refactor Agent).

### 4.4 Repository Insights View
- **Overview Tab:**
  - Health score (/100).
  - Tech-debt item count.
  - Test coverage percentage.
  - Repository summary generated by AI.
  - Language distribution progress bars.
  - Recent agent activity log (e.g., "Refactor Agent detected 4 duplicates").
- **Chat Tab:**
  - Conversational interface to query the codebase.
  - AI responses include citations and file paths mapped to exact lines of code.
  - Predefined templates (e.g., "Explain the authentication flow").
- **Architecture Tab:**
  - Visual node-based dependency graph mapping the system's architecture (Client, API Gateway, Auth Service, Database, external APIs like Stripe).
  - Breakdown of system layers (Transport, Domain, Persistence).
  - Hot paths mapping API routes to services.
- **Docs & Tech Debt Tabs:**
  - Dedicated sections for auto-generated documentation and identifying specific code segments that require refactoring.



## 5. Current State & Limitations
- The project is primarily a **frontend implementation** with mocked data (e.g., `src/lib/mock-data.ts`).
- The actual analysis engine (AST parsing, embeddings generation, multi-agent AI execution) needs to be implemented on the backend.
- Authentication and database connections (e.g., to store historical analysis and user sessions) are pending integration.

## 6. Future Roadmap
1. **Backend Integration:** Implement real ingestion pipelines using Git cloning, Tree-sitter for AST parsing, and an LLM API (e.g., OpenAI or Anthropic) for generation.
2. **Vector Database Setup:** Introduce a vector database (e.g., Pinecone, Weaviate, or pgvector) for semantic code search.
3. **Authentication:** Add user authentication (e.g., via Clerk, Auth0, or Supabase).
4. **Private Repositories:** Add GitHub App integration to support OAuth and private repository access.
